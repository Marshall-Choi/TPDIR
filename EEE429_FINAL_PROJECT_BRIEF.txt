--------------------------------------------------------------------------------
1. 목표 (Objectives)
--------------------------------------------------------------------------------
- Pynq-Z2 보드에서 MNIST CNN 추론 파이프라인을 최적화할 것.
- 하드웨어 추론 정확도는 최소 93% 이상.


--------------------------------------------------------------------------------
2. 모델 개요 (수학적 동등성 유지 — 구조 변경 금지)
--------------------------------------------------------------------------------
MNIST (1×28×28)
  → Conv2d 1→16, kernel 3×3  → ReLU
  → Conv2d 16→32, kernel 3×3 → ReLU
  → Conv2d 32→32, kernel 3×3 → ReLU
  → MaxPool2d 2×2
  → Flatten 후 FC: 3872 → 10
  → 클래스 라벨

특징맵 크기 참고:
  28→26→24→22 → pool → 11×11×32 = 3872


--------------------------------------------------------------------------------
3. 실행/커널 개요
--------------------------------------------------------------------------------
1) .npy 가중치/바이어스 등 로드
2) Overlay(비트스트림) 로드
3) 이미지 전처리 (선택)
4) 하드웨어 커널 호출 루프 (↔ 누적)
5) 예측 결과 누적
6) 정확도 계산

런타임 측정에 포함되는 구간 (보고 시 이 세 구간만):
  - Image Preprocessing
  - Hardware Kernel Calling Loop
  - Accumulating Prediction Result


--------------------------------------------------------------------------------
4. 규칙 (Rules)
--------------------------------------------------------------------------------
[금지]
- 수학적으로 동등하지 않은 방식으로 NN 아키텍처 변경
- 드라이버 스크립트에서 numpy, pynq, 내장 모듈 **이외** 외부 모듈 import
- NN의 일부만 하드웨어에 두고 나머지는 소프트웨어로 처리 (전체를 HW에 구현)
- 본인이 작성하지 않은 소스 그대로 사용
- DDR 타이밍 파라미터 수정

[허용]
- 양자화기(quantizer) 또는 양자화 방식 최적화
- QAT(Quantization-Aware Training)로 재학습

※ 해석: PC에서 PyTorch 등으로 학습·양자화·.npy 추출을 하는 것은 일반적으로
  “드라이버 스크립트” 제한과 별개(보드에서 돌리는 호스트 코드가 제한 대상).
  제출 Notebook은 교수 과제 정의에 맞게 numpy/pynq만 쓰는지 반드시 확인할 것.
